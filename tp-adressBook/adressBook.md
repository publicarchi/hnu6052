# Définir un modèle de document

Maintenant que nous avons pris en main l’éditeur oXygen et que nous savons éditer des fichiers XML bien formés, je vous propose de créer votre premier modèle de document XML.

Le cas d’usage auquel vous devez répondre est un cas de figure relativement classique dans la production de données structurées. Il s’agit de définir un modèle de document pour gérer les informations d’un carnet d’adresse.

À partir de vos connaissances actuelles de XML, il s’agit donc de définir la meilleure utilisation possible des éléments XML est des attributs pour créer une structure de données efficace afin de traiter les données d’un fichier de personnes.

Ce répertoire doit comprendre, les noms et prénoms des individus, leur adresse et coordonnées téléphoniques et courriel.

## Pas à pas

- Identification des informations à traiter
- Quelles implémentations possibles en XML
- Choix du nom des éléments
- Regroupement des informations sur une personne
- Utilisation de la hiérarchie
- Quand utiliser des éléments ou des attributs
- Description de la structure du document (élément racine, éléments fils, etc.)
- Contrôle de la saisie à l’aide d’un schema
